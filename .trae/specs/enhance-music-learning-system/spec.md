# 增强音乐学习系统 Spec

## Why
当前的音乐学习应用基础教学环节过于简单（例如仅敲击一次按键即可通过），这不符合音乐教育中建立肌肉记忆和循序渐进的逻辑。我们需要一个更系统化的基础教学流程，在用户进入更复杂的“游戏模式”或曲目演奏前，确保他们通过反复练习和模式组合打牢基础。

## What Changes
- 修改关卡数据模型（`Level` 接口），支持定义 `requiredHits`（需要连续/累计正确弹奏的次数）或 `sequence`（需要按顺序弹奏的音符序列），而不仅仅是单次按键。
- 重构教学关卡（Teaching Level）逻辑，要求用户多次正确输入同一音符或完成一组音符序列才能通关。
- 增加关卡内的进度反馈 UI（例如：“已完成 1/5 次”或进度条），让用户明确当前练习的目标。
- 重新设计 `src/data/levels.ts` 中的初期关卡，使其符合音乐教学逻辑（例如：学习 C4 -> 练习 C4 5次 -> 学习 D4 -> 练习 D4 5次 -> 混合练习 C4 和 D4 -> 学习 E4...）。
- **BREAKING**: `Level` 接口结构将发生变化，涉及解析该接口的页面（如 `LevelPlay.tsx`）需要同步更新。

## Impact
- Affected specs: 关卡进度系统、教学与测试关卡逻辑实现
- Affected code: `src/data/levels.ts`, `src/pages/LevelPlay.tsx`, `src/store/useStore.ts`

## ADDED Requirements
### Requirement: 关卡内进度与重复练习机制
系统应在基础教学和单音测试关卡中，要求用户对同一音符进行多次正确输入，或完成特定序列的输入，以建立肌肉记忆。

#### Scenario: 基础教学重复练习
- **WHEN** 用户进入“学习 C4 (Do)”的教学关卡
- **THEN** 界面提示用户需要弹奏 C4 键 5 次。
- **WHEN** 用户正确弹奏一次 C4
- **THEN** 界面进度更新为 1/5，并要求继续弹奏。
- **WHEN** 用户完成第 5 次正确弹奏
- **THEN** 关卡才算通过，结算弹窗出现。

### Requirement: 循序渐进的混合测试模式
在学习了多个新音符后，系统应提供混合练习，巩固音符间的相对位置关系。

#### Scenario: 混合音符练习
- **WHEN** 用户学习完 C4 和 D4
- **THEN** 下一个关卡是 C4 与 D4 的交替或随机序列练习（例如 C-D-C-D），用户必须完整且正确地弹奏该序列才能通关。

## MODIFIED Requirements
### Requirement: 教学关卡 (Teaching Level) 通关条件
**原逻辑**：敲击一次目标音符即通关。
**新逻辑**：根据关卡配置的 `requiredHits` 或 `sequence`，完成所有要求的按键次数/序列后才可通关。若在此过程中出错，可根据设计重置当前进度或仅扣除得分（基础阶段建议强制要求正确率或不重置但要求补足次数）。

## REMOVED Requirements
### Requirement: 单次按键通关
**Reason**: 无法有效建立肌肉记忆，违背音乐教学常理。
**Migration**: 所有原有的“单音测试”或“教学”关卡将默认要求至少 3-5 次的连续或累计正确弹奏。
