# 键盘美化与弹窗逻辑修复 Spec

## Why
目前在 `regression_test` 等关卡中，“关卡完成！”弹窗依然会打断用户体验。如果希望在非最终挑战关卡都保持连贯，我们需要审查所有弹窗逻辑或开启全局 `autoNext` 机制。另外，目前虚拟键盘黑键的紫色边框与整体青蓝色（Cyan）的科幻/科技感主题不协调，显得突兀和杂乱。我们需要对键盘进行视觉美化，统一色彩体系。

## What Changes
- **统一键盘配色**：移除黑键突兀的紫色 (`fuchsia`) 边框与发光效果。统一使用同色系（如深蓝/青蓝/灰黑）来区分黑白键，使键盘更具质感。目标键提示颜色可以保留醒目的红色，按下反馈保留绿色，以保持强烈的状态对比。
- **弹窗逻辑修正**：确保除了 `song` (曲目) 或需要明显结算的节点外，其他所有过程测试（包括 `regression_test` 回归测试关卡）的 `autoNext` 属性都设置为 `true`，以实现真正的一镜到底无缝学习。
- **弹窗视觉优化**（可选）：使得确实需要显示的弹窗设计得更加轻量。

## Impact
- Affected specs: UI 美化、关卡流转体验
- Affected code: `src/components/Keyboard.tsx`, `src/data/levels.ts`

## ADDED Requirements
### Requirement: 键盘视觉统一
虚拟键盘的黑键不应该使用偏离主色调（紫色/洋红）的颜色，应采用与背景（深灰/黑）和主色调（青蓝）相协调的设计。

#### Scenario: 键盘渲染
- **WHEN** 渲染黑键时
- **THEN** 黑键的边框或阴影应该使用深青色或浅灰色，而不是原本的紫色。

## MODIFIED Requirements
### Requirement: 回归测试的无缝流转
回归测试在通过后，不应显示完整的结算模态框，而是自动流转到奖励曲目或下一个学习阶段。

#### Scenario: 回归测试成功
- **WHEN** 用户通过了“阶段一回归测试 (C,D,E)”
- **THEN** 不弹出“关卡完成！”弹窗，而是倒计时1.5秒后直接进入“简单曲段”关卡。

## REMOVED Requirements
- 移除了所有黑键的 `fuchsia-*` Tailwind 类。
