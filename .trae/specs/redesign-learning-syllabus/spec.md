# 重新梳理教学大纲 Spec

## Why
当前的音乐学习大纲跳跃过大且缺乏连贯性。对于零基础用户，缺乏最基本的按键与键盘映射认知；在学习过程中频繁的弹窗打断了“心流”；同时，缺乏基于传统音乐教学体系（如：认音->找键->单音练习->音阶->乐句->曲目）的科学编排。我们需要设计一个平滑、系统、不被打断的线上音乐教学大纲，提供零基础入门，并实现自动连贯的教学体验。

## What Changes
- **深度梳理线上音乐教学大纲**：
  - **阶段 0（破冰与指引）**：认识电脑键盘与虚拟钢琴键盘的映射关系（引导用户找到“A”键对应“C4”）。
  - **阶段 1（基础单音与相邻音）**：学习 C4 (Do), D4 (Re), E4 (Mi)。每个音符包含“认音（提示按哪个键） -> 盲按练习 -> 混合练习”。
  - **阶段 2（音阶雏形与乐句）**：学习 F4 (Fa), G4 (Sol)。加入简单的五声音阶上行/下行练习。
  - **阶段 3（综合运用与小曲）**：加入《欢乐颂》片段、《小星星》等。
- **自动连贯学习（无缝过关）**：在“基础教学”和“连续练习”的关卡组中，取消成功后的结算弹窗，改为界面上简单的“通关特效”，并**自动倒计时 1~2 秒后直接进入下一关**，保持用户的学习连贯性。
- **键盘视觉引导增强**：在“教学”阶段或用户长时间未操作/按错时，界面上的虚拟键盘不仅要高亮目标音符，还应该有一个**闪烁或高亮动画的提示**，明确告诉用户“现在该敲击这个键”。

## Impact
- Affected specs: 关卡系统、关卡游玩页面 (`LevelPlay.tsx`)、键盘组件 (`Keyboard.tsx`)
- Affected code: `src/data/levels.ts`, `src/pages/LevelPlay.tsx`, `src/components/Keyboard.tsx`

## ADDED Requirements
### Requirement: 零基础键盘映射引导
系统必须在第一关明确告诉用户电脑键盘（如字母 A）与钢琴按键（C4）的对应关系，并在虚拟键盘上用醒目的方式（如文字标签）标出对应的电脑按键。

#### Scenario: 认识键盘
- **WHEN** 用户进入第 1 关
- **THEN** 界面上 C4 琴键上闪烁显示字母“A”，提示文字为：“请按下键盘上的字母 A 来弹奏 Do (C4)”。

### Requirement: 关卡无缝衔接
为了不打扰学习心流，对于同一学习阶段内的连贯关卡，过关后不应强制用户点击“继续”，而是自动过渡。

#### Scenario: 自动进入下一关
- **WHEN** 用户在“练习 C4”关卡中完成了 5 次击键
- **THEN** 界面显示“太棒了！接下来学习 D4”，并在 1.5 秒后自动切换到“学习 D4”关卡的逻辑，无需用户点击按钮。

### Requirement: 动态视觉引导提示
在教学关卡中，目标按键应有持续的呼吸灯/闪烁效果以吸引视线。

#### Scenario: 目标键闪烁
- **WHEN** 轮到用户弹奏 E4 时
- **THEN** 虚拟键盘上的 E4 按键呈现呼吸发光效果，直到用户按下正确的键。

## MODIFIED Requirements
### Requirement: 关卡列表数据
更新 `LEVELS` 数组。新的结构将按照：
1. 学习 C4 -> 2. 练习 C4 -> 3. 学习 D4 -> 4. 练习 D4 -> 5. 混合 C4-D4 -> 6. 学习 E4 -> 7. 练习 E4 -> 8. 阶段一回归测试 (C,D,E) -> 9. 简单曲段(C-D-E-C) -> 10. 学习 F4 -> 11. 练习 F4 -> 12. 学习 G4 -> 13. 练习 G4 -> 14. 阶段二回归测试 (C,D,E,F,G) -> 15. 欢乐颂(E-E-F-G-G-F-E-D-C-C-D-E-E-D-D) -> 16. 学习 A4 -> 17. 练习 A4 -> 18. 阶段三回归测试 (C~A) -> 19. 曲目挑战：小星星 -> 20. 智能复习

## REMOVED Requirements
### Requirement: 旧版跳跃式大纲
**Reason**: 导致新手挫败，不符合教育逻辑。
**Migration**: 彻底替换原有的前 9 关，采用新的 20 关结构。
