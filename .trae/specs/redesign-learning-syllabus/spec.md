# 重新梳理教学大纲 Spec

## Why
当前的音乐学习大纲跳跃过大。用户仅学习了 C4、D4 和 E4 后，立刻被要求弹奏《小星星》，而《小星星》的音符包含了 F4、G4 甚至 A4。这种难度突增会让用户感到挫败。我们需要设计一个平滑、系统、循序渐进的教学大纲，在挑战曲目前确保所有必要的基础音符都被充分学习和测试。

## What Changes
- 重新设计 `src/data/levels.ts`，将关卡扩充为包含完整的 C大调核心音阶（C4~A4）的系统化学习路线。
- 引入“学习 -> 练习 -> 混合练习 -> 回归测试”的闭环，在每个闭环后加入一个非常简单的小曲段作为正向反馈。
- 新增大纲结构：
  - 阶段1：学习 C4 (Do), D4 (Re), E4 (Mi)，并进行阶段回归测试。
  - 阶段1奖励：极简练习曲（例如《欢乐颂》开头片段 E-E-F-G 简化版，或者 C-D-E-C 小片段）。
  - 阶段2：学习 F4 (Fa), G4 (Sol)，并进行阶段回归测试。
  - 阶段2奖励：《欢乐颂》完整乐句。
  - 阶段3：学习 A4 (La)，并进行阶段回归测试。
  - 最终挑战：《小星星》第一句。

## Impact
- Affected specs: 关卡系统
- Affected code: `src/data/levels.ts`
- 关卡总数将增加，为确保旧用户的兼容性，新增加的关卡可能会影响现有的解锁进度（不过作为测试期应用，直接覆盖数据是可以接受的）。

## ADDED Requirements
### Requirement: 平滑过渡的教学曲线
系统在要求用户弹奏一首包含多个音符的曲目前，必须确保曲目中的所有音符都已经被单独“学习”、“练习”并通过了“回归测试”。

#### Scenario: 弹奏曲目前的基础铺垫
- **WHEN** 用户在关卡列表中看到“曲目挑战：小星星”
- **THEN** 该关卡前必定有涵盖 C4、D4、E4、F4、G4、A4 的学习、练习与测试关卡，且用户必须按顺序通过这些前置关卡。

## MODIFIED Requirements
### Requirement: 关卡列表数据
更新 `LEVELS` 数组。新的结构将按照：
1. 学习 C4 -> 2. 练习 C4 -> 3. 学习 D4 -> 4. 练习 D4 -> 5. 混合 C4-D4 -> 6. 学习 E4 -> 7. 练习 E4 -> 8. 阶段一回归测试 (C,D,E) -> 9. 简单曲段(C-D-E-C) -> 10. 学习 F4 -> 11. 练习 F4 -> 12. 学习 G4 -> 13. 练习 G4 -> 14. 阶段二回归测试 (C,D,E,F,G) -> 15. 欢乐颂(E-E-F-G-G-F-E-D-C-C-D-E-E-D-D) -> 16. 学习 A4 -> 17. 练习 A4 -> 18. 阶段三回归测试 (C~A) -> 19. 曲目挑战：小星星 -> 20. 智能复习

## REMOVED Requirements
### Requirement: 旧版跳跃式大纲
**Reason**: 导致新手挫败，不符合教育逻辑。
**Migration**: 彻底替换原有的前 9 关，采用新的 20 关结构。
